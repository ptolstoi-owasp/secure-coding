# 🛡️ Secure Coding Guidelines & Dokumentation

Willkommen im **Secure Coding** Repository! 

Dieses Repository dient als zentraler Wissensspeicher und Dokumentations-Hub für sichere Softwareentwicklung. Ziel ist es, Entwicklern praxisnahe Richtlinien an die Hand zu geben, um moderne Webapplikationen effektiv gegen die häufigsten Angriffsvektoren abzusichern.

## 📚 Verfügbare Dokumente

Unser Hauptdokument basiert auf den aktuellen **OWASP Top 10 (2025)** und bietet konkrete Anti-Patterns sowie sichere Lösungsansätze, primär fokussiert auf C# (ASP.NET Core) und TypeScript (Node.js/React).

👉 **[📖 Zum OWASP Praxisleitfaden für Entwickler (Deutsch)](OWASP-Praxisleitfaden.md)**

## 💻 Code-Beispiele & Applikationen

*Hinweis:* Um Theorie und Praxis sauber zu trennen, enthält dieses Repository **ausschließlich Dokumentationen und Richtlinien**. 

Ein separates Repository mit ausführbaren Demo-Applikationen und vollständigen Code-Beispielen zu diesen Richtlinien befindet sich aktuell im Aufbau und wird hier in Kürze verlinkt.

## 🤝 Mitwirken (Contributing)

Sicherheit ist ein stetiger Prozess. Wenn Du Ergänzungen hast, Fehler im Leitfaden findest oder Best Practices für weitere Frameworks (wie z. B. Java Spring Boot, Python Django) hinzufügen möchtest:
1. Erstelle einen Fork dieses Repositories.
2. Reiche Deine Änderungen als Pull Request (PR) ein.

## 🔒 Sicherheit (Security)

Falls Du beim Lesen unserer Richtlinien oder in unseren zukünftigen Code-Beispielen eine Sicherheitslücke entdeckst, bitten wir Dich, diese nicht öffentlich zu machen. Bitte lies dazu unsere [Sicherheitsrichtlinie (SECURITY.md)](SECURITY.md) für Anweisungen zur verantwortungsvollen Offenlegung (Responsible Disclosure).
